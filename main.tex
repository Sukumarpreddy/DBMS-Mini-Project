\section*{Table Creation}

\subsection*{STUDENT}
\begin{lstlisting}[language=SQL]
CREATE TABLE STUDENT (
    name VARCHAR(50),
    snumber INT PRIMARY KEY,
    class VARCHAR(20),
    major VARCHAR(20)
);
\end{lstlisting}

\subsection*{COURSE}
\begin{lstlisting}[language=SQL]
CREATE TABLE COURSE (
    cname VARCHAR(50),
    cnumber INT PRIMARY KEY,
    credit_hrs INT,
    dept VARCHAR(20)
);
\end{lstlisting}

\subsection*{SECTION}
\begin{lstlisting}[language=SQL]
CREATE TABLE SECTION (
    sectionid INT PRIMARY KEY,
    cnumber INT,
    sem VARCHAR(20),
    year INT,
    instructor VARCHAR(50),
    FOREIGN KEY (cnumber)
    REFERENCES COURSE(cnumber)
);
\end{lstlisting}


\subsection*{GRADE\_REPORT}
\begin{lstlisting}[language=SQL]
CREATE TABLE GRADE_REPORT (
    snumber INT,
    sectionid INT,
    grade CHAR(2),
    FOREIGN KEY (snumber)
    REFERENCES STUDENT(snumber),
    FOREIGN KEY (sectionid)
    REFERENCES SECTION(sectionid)
);
\end{lstlisting}

\section*{Question 1}

\textbf{Retrieve the names of all senior students majoring in 'cs'.}

\vspace{0.4cm}

\textbf{Explanation}

The STUDENT table contains information about students, including their name, student number, class, and major. In many academic databases, it is often necessary to identify students belonging to a particular class and academic discipline for reporting and analysis purposes.

This query retrieves the names of students who satisfy two specific conditions. The first condition requires that the student belongs to the \textbf{senior} class, which generally represents students in the final stage of their academic program. The second condition requires that the student's major is \textbf{cs} (Computer Science).

The \textbf{SELECT} statement is used to retrieve the required attribute, namely the student's name. The \textbf{FROM} clause specifies that the data is to be obtained from the STUDENT table. The \textbf{WHERE} clause applies filtering conditions to the records. The logical operator \textbf{AND} ensures that only those records satisfying both conditions simultaneously are selected.

When the query is executed, the database scans the STUDENT table, evaluates each row against the specified conditions, and returns the names of all matching students. This type of query is commonly used for generating departmental reports, identifying eligible students for projects, and preparing academic statistics.

\vspace{0.4cm}

\textbf{SQL Query}

\begin{lstlisting}[language=SQL]
SELECT name
FROM STUDENT
WHERE class = 'senior'
AND major = 'cs';
\end{lstlisting}

\hline
Rahul \\
\hline
Vikas \\
\hline
\end{tabular}
\end{center}

\vspace{0.7cm}

\textbf{Result}

The query was executed successfully and returned the names of students who belong to the \textbf{senior} class and are majoring in \textbf{Computer Science}. Based on the data available in the STUDENT table, the students \textbf{Rahul} and \textbf{Vikas} satisfy both conditions. Therefore, their names are displayed in the result. The output confirms that the filtering conditions specified in the WHERE clause have been applied correctly and that only the required records have been retrieved from the database.

\section*{Question 2}

\textbf{Retrieve the names of all courses taught by Professor King in 2016 and 2017.}

\vspace{0.4cm}

\textbf{Explanation}

The information about courses is stored in the \textbf{COURSE} table, while details regarding course offerings, instructors, semesters, and academic years are maintained in the \textbf{SECTION} table. To determine the courses taught by Professor King during the years 2016 and 2017, it is necessary to combine information from both tables.

The query uses the \textbf{JOIN} operation to connect the COURSE and SECTION tables through the common attribute \textbf{cnumber}. This allows course names to be matched with their corresponding section details. The \textbf{WHERE} clause filters the records to include only those sections taught by the instructor \textbf{'king'} and offered during the years \textbf{2016} or \textbf{2017}.

The \textbf{IN} operator is used to specify multiple years within a single condition. The \textbf{DISTINCT} keyword ensures that duplicate course names are removed from the result, so each course appears only once even if it was taught in multiple sections. After applying these conditions, the query returns the names of all courses taught by Professor King during the specified academic years.

\vspace{0.4cm}

\textbf{SQL Query}

\begin{lstlisting}[language=SQL]
SELECT DISTINCT C.cname
FROM COURSE C
JOIN SECTION S
ON C.cnumber = S.cnumber
WHERE S.instructor = 'king'
AND S.year IN (2016, 2017);
\end{lstlisting}

\vspace{0.5cm}

\textbf{Output}

\vspace{0.3cm}

\begin{center}
\renewcommand{\arraystretch}{1.6}
\begin{tabular}{|c|}
\hline
\textbf{CNAME} \\
\hline
DBMS \\
\hline
OS \\
\hline
\end{tabular}
\end{center}

\vspace{0.7cm}

\textbf{Result}

The query was executed successfully and retrieved the names of all courses taught by Professor King during the academic years 2016 and 2017. By joining the COURSE and SECTION tables and applying the specified filtering conditions, the courses \textbf{DBMS} and \textbf{OS} were identified. The use of the DISTINCT keyword ensured that each course name appeared only once in the final result.

\section*{Question 3}

\textbf{For each section taught by Professor King, retrieve the course number, semester, year, and number of students who took the section.}

\vspace{0.4cm}

\textbf{Explanation}

The SECTION table contains details about course sections, including the course number, semester, year, and instructor. The GRADE\_REPORT table stores information about students enrolled in different sections. To determine how many students attended each section taught by Professor King, data from both tables must be combined.

The query uses a \textbf{LEFT JOIN} operation between the SECTION and GRADE\_REPORT tables based on the common attribute \textbf{sectionid}. A LEFT JOIN is used to ensure that all sections taught by Professor King are included in the result, even if no students are enrolled in a particular section.

The \textbf{WHERE} clause filters the records to include only those sections taught by the instructor \textbf{'king'}. The \textbf{COUNT()} aggregate function is then used to count the number of student numbers associated with each section. To calculate the count separately for every section, the \textbf{GROUP BY} clause groups records according to section identifier, course number, semester, and year.

As a result, the query provides a summary showing the course number, semester, academic year, and the total number of students enrolled in each section conducted by Professor King.

\vspace{0.4cm}

\textbf{SQL Query}

\begin{lstlisting}[language=SQL]
SELECT S.cnumber,
       S.sem,
       S.year,
       COUNT(G.snumber) AS number_of_students
FROM SECTION S
LEFT JOIN GRADE_REPORT G
ON S.sectionid = G.sectionid
WHERE S.instructor = 'king'
GROUP BY S.sectionid,
         S.cnumber,
         S.sem,
         S.year;
\end{lstlisting}

\vspace{0.5cm}

\textbf{Output}

\vspace{0.3cm}

\begin{center}

\begin{tabular}{|c|c|c|c|}

\textbf{SEM} &
\textbf{YEAR} &
\textbf{NUMBER\_OF\_STUDENTS} \\
\hline
101 & Odd  & 2016 & 2 \\
\hline
102 & Even & 2017 & 1 \\
\hline
\end{tabular}
\end{center}

\vspace{0.7cm}


\textbf{Result}

The query was executed successfully and retrieved the number of students enrolled in each section taught by Professor King.

\section*{Question 4}

\textbf{Explanation}

The STUDENT table contains information about students, including their names and student numbers. The GRADE\_REPORT table stores the grades obtained by students in various sections. To display student names along with their corresponding grades, the two tables must be connected using the common attribute \textbf{snumber}.

The query retrieves the student's name from the STUDENT table and the grade from the GRADE\_REPORT table. The \textbf{WHERE} clause establishes the relationship between the tables by matching the student numbers. In this query, the objective is to identify courses that have more than 3 credit hours. The \textbf{SELECT} statement is used to retrieve the course name (\textbf{cname}) and the corresponding credit hours (\textbf{credit\_hrs}). The \textbf{WHERE} clause applies a condition to filter only those records where the value of credit hours is greater than 3.

The database examines each record in the COURSE table and compares the credit hour value against the specified condition. Courses satisfying the condition are included in the result set, while all other courses are excluded. The GRADE\_REPORT table stores the grades obtained by students in different course sections. To identify students who have secured a particular grade, information from both tables must be combined using the common attribute \textbf{snumber}.

The query joins the STUDENT and GRADE\_REPORT tables by matching the student numbers present in both tables. The \textbf{WHERE} clause is used to establish the relationship between the tables and to filter the records based on the specified grade value. In this case, only those records where the grade is equal to \textbf{'A'} are selected. The \textbf{SELECT} statement retrieves the names of the students who satisfy this condition.

This type of query is useful for identifying high-performing students, preparing merit lists, generating academic reports, and analyzing student performance. \section*{Question 7}

\textbf{Display the number of students in each class.}

\vspace{0.4cm}

\textbf{Explanation}

The STUDENT table stores information about students, including their name, student number, class, and major. Educational institutions often require summarized information about the distribution of students across different classes for academic planning and administrative purposes.

The objective of this query is to determine the total number of students belonging to each class category. To achieve this, the query uses the \textbf{COUNT()} aggregate function, which counts the number of records present in each group. The \textbf{GROUP BY} clause is used to group all students according to their class value, such as junior or senior.

When the query is executed, the database scans all records in the STUDENT table and organizes them into separate groups based on the class attribute. The COUNT() function then calculates the number of students present in each group. The resulting output displays the class name along with the corresponding number of students.

Such aggregate queries are widely used in database systems to generate statistical summaries, prepare reports, and analyze data efficiently. Instead of displaying individual student records, the query provides a concise overview of student distribution among different classes.

\vspace{0.4cm}

\textbf{SQL Query}

\begin{lstlisting}[language=SQL]
SELECT class,
       COUNT(*) AS total_students
FROM STUDENT
GROUP BY class;
\end{lstlisting}

\vspace{0.5cm}

\textbf{Output}

\vspace{0.3cm}

\begin{center}
\renewcommand{\arraystretch}{1.6}
\begin{tabular}{|c|c|}
\hline
\textbf{CLASS} & \textbf{TOTAL\_STUDENTS} \\
\hline
senior & 2 \\
\hline
junior & 1 \\
\hline
\end{tabular}
\end{center}

\vspace{0.7cm}

\textbf{Result}

The query was executed successfully and displayed the total number of students belonging to each class category. The output shows that there are \textbf{2 students} in the \textbf{senior} class and \textbf{1 student} in the \textbf{junior} class. This confirms that the COUNT() aggregate function and GROUP BY clause were applied correctly to summarize the student data. The generated result provides a clear overview of the distribution of students across different classes and demonstrates the use of aggregation operations in SQL for producing meaningful statistical information.
